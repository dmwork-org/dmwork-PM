# 运行时视图

> 三个关键运行时场景：用户发消息给 Bot、Bot 流式回复、群聊 @mention 处理。

## 概述

本文档通过时序图描述三个核心运行时场景，展示消息在 Octo + OpenClaw 生态中的完整流转路径。

---

## 场景 1：用户发消息给 Bot（完整主流程）

这是最核心的场景，涵盖从用户发消息到 AI 回复的完整链路。

### 参与者

| 参与者 | 角色 |
|--------|------|
| 👤 Human User | 人类用户 |
| 📱 DMWork Client | Web/iOS/Android 客户端 |
| 🔌 WuKongIM | 底层 IM 通讯引擎 |
| 🖥️ DMWork Server | dmworkim 业务服务器 |
| 🔴 Redis | 缓存和事件队列 |
| 🐬 MySQL | 数据库 |
| 🦞 DMWork Plugin | openclaw-channel-dmwork |
| 🔒 WKSocket | WuKongIM WebSocket 客户端（DH+AES 加密） |
| 🎛️ OpenClaw Gateway | AI 助手控制面 |
| 🧠 LLM Provider | Claude/GPT/Gemini 等 |

### Phase 0：系统启动（Bot 注册 & WebSocket 连接）

```mermaid
sequenceDiagram
    participant GW as 🎛️ Gateway
    participant Plugin as 🦞 Plugin
    participant DMServer as 🖥️ DMServer
    participant MySQL as 🐬 MySQL
    participant WKIM as 🔌 WuKongIM
    participant WKSock as 🔒 WKSocket

    GW->>Plugin: 加载插件，调用 startAccount()
    Plugin->>DMServer: POST /v1/bot/register {botToken: "bf_xxx"}
    DMServer->>MySQL: 查询 Bot 信息 & 验证
    MySQL-->>DMServer: Bot 记录（robot_id, app_id）
    DMServer->>WKIM: 注册 IM 用户，获取 Token
    WKIM-->>DMServer: im_token + ws_url
    DMServer-->>Plugin: {robot_id, im_token, ws_url, owner_uid}
    
    Plugin->>WKSock: 创建 WKSocket 实例
    WKSock->>WKIM: WebSocket 连接（ws_url）
    WKSock->>WKIM: CONNECT 包（uid, token, clientKey=DH公钥）
    WKIM-->>WKSock: CONNACK（serverKey, salt）
    Note over WKSock: DH 密钥交换：<br/>aesKey = MD5(base64(DH(priv,pub)))[0:16]<br/>aesIV = salt[0:16]
    
    WKSock->>WKSock: 启动心跳定时器（60s）
    Plugin->>Plugin: 启动 HTTP 心跳（定期 POST /v1/bot/heartbeat）
```

### Phase 1-2：用户发消息 → WuKongIM 投递给 Bot

```mermaid
sequenceDiagram
    participant User as 👤 用户
    participant Client as 📱 客户端
    participant WKIM as 🔌 WuKongIM
    participant WKSock as 🔒 WKSocket
    participant Plugin as 🦞 Plugin

    User->>Client: 输入并发送消息
    Client->>WKIM: SEND 包（WebSocket, AES-CBC 加密 payload）
    WKIM-->>Client: SENDACK（messageID, messageSeq）
    Client->>Client: UI 显示已发送 ✓
    
    Note over WKIM: 持久化消息 → 查找订阅者 → 投递
    
    WKIM->>WKSock: RECV 包（fromUID, channelID, payload=AES加密）
    WKSock->>WKSock: AES-CBC 解密 → JSON payload
    WKSock->>WKIM: RECVACK（messageID，确认收到）
    WKSock->>Plugin: onMessage 回调（BotMessage）
```

### Phase 3：业务层并行处理（DMWork Server）

```mermaid
sequenceDiagram
    participant WKIM as 🔌 WuKongIM
    participant DMServer as 🖥️ DMServer
    participant MySQL as 🐬 MySQL
    participant Redis as 🔴 Redis

    WKIM->>DMServer: robotMessageListen gRPC 回调
    DMServer->>DMServer: 解析 payload，识别目标 Bot
    DMServer->>MySQL: 查询 Bot（有 Redis 24h 缓存）
    MySQL-->>DMServer: Bot 存在 ✓
    DMServer->>MySQL: 检查好友关系
    MySQL-->>DMServer: 是好友 ✓
    DMServer->>Redis: ZADD robotEvent:{robotID}（存储消息事件）
    DMServer->>WKIM: IMClearConversationUnread（清除 Bot 未读数）
```

### Phase 4-5：插件处理 + 已读/typing

```mermaid
sequenceDiagram
    participant Plugin as 🦞 Plugin
    participant DMServer as 🖥️ DMServer
    participant GW as 🎛️ Gateway

    Plugin->>Plugin: 过滤检查（跳过自己发的/其他Bot/不支持类型）
    Plugin->>Plugin: resolveContent（解析文本/图片/文件/语音）
    Plugin->>Plugin: 构建 InboundContext（Body, MediaUrl, SessionKey...）
    Plugin->>GW: recordInboundSession（持久化会话元数据）
    
    par 并行
        Plugin->>DMServer: POST /v1/bot/readReceipt（已读回执）
        Note over DMServer: WuKongIM 推送已读给客户端 ✓✓
    and
        Plugin->>DMServer: POST /v1/bot/typing（输入状态）
        Note over DMServer: 客户端显示"正在输入..." 💬
    end
    
    Plugin->>Plugin: 启动 typing 定时器（每 5s 续期）
```

### Phase 6-7：AI 推理 + 流式回复

```mermaid
sequenceDiagram
    participant Plugin as 🦞 Plugin
    participant GW as 🎛️ Gateway
    participant LLM as 🧠 LLM
    participant DMServer as 🖥️ DMServer
    participant Client as 📱 客户端

    Plugin->>GW: dispatchReplyWithBufferedBlockDispatcher()
    GW->>GW: 加载 Agent 配置（SOUL.md, MEMORY.md 等）
    GW->>LLM: Chat Completion API（stream: true）
    
    loop 流式 Token 输出
        LLM-->>GW: SSE delta chunk
        GW-->>Plugin: onPartialReply({text})
        
        alt 首次 partial
            Plugin->>DMServer: POST /v1/bot/stream/start {payload}
            DMServer-->>Plugin: {stream_no}
            Client->>Client: UI 创建流式消息气泡
        else 后续 partial
            Plugin->>DMServer: POST /v1/bot/sendMessage {stream_no, ...}
            Client->>Client: UI 实时更新消息内容 ▊
        end
    end
    
    LLM-->>GW: [DONE]
    GW-->>Plugin: deliver({text: 最终完整文本})
    
    Plugin->>Plugin: 停止 typing 定时器
    Plugin->>DMServer: POST /v1/bot/stream/end {stream_no}
    Client->>Client: UI 冻结最终消息 ✅
```

## 场景 2：Bot 流式回复（详细）

聚焦流式消息协议的状态机：

```mermaid
stateDiagram-v2
    [*] --> Idle: 等待消息
    Idle --> Processing: 收到用户消息
    Processing --> TypingSent: 发送 typing 状态
    TypingSent --> StreamStarted: POST /v1/bot/stream/start → 获取 stream_no
    StreamStarted --> Streaming: 首次 LLM token 到达
    Streaming --> Streaming: 后续 token → POST sendMessage(stream_no)
    Streaming --> StreamEnded: LLM [DONE] → POST /v1/bot/stream/end
    StreamEnded --> Idle: 回复完成
    
    Streaming --> FallbackDelivery: 流式失败（streamFailed=true）
    FallbackDelivery --> Idle: 一次性发送完整文本
    
    Processing --> ErrorResponse: AI 推理失败
    ErrorResponse --> Idle: 发送错误提示消息
```

### 流式消息关键参数

```
POST /v1/bot/stream/start
Body: {
  channel_id: "user_uid",
  channel_type: 1,           // 1=DM, 2=群聊
  payload: base64({
    type: 1,                 // ContentType: Text
    content: "首次文本"
  })
}
Response: { stream_no: "xxx-yyy" }

---

POST /v1/bot/sendMessage（流式续传）
Body: {
  channel_id: "user_uid",
  channel_type: 1,
  stream_no: "xxx-yyy",      // 关键：携带 stream_no
  payload: {
    type: 1,
    content: "累积文本..."    // 每次发完整累积文本（非 delta）
  }
}

---

POST /v1/bot/stream/end
Body: {
  stream_no: "xxx-yyy",
  channel_id: "user_uid",
  channel_type: 1
}
```

## 场景 3：群聊 @mention 处理

当用户在群聊中 @Bot 时的处理流程：

```mermaid
sequenceDiagram
    participant User as 👤 用户
    participant WKIM as 🔌 WuKongIM
    participant WKSock as 🔒 WKSocket
    participant Plugin as 🦞 Plugin
    participant DMServer as 🖥️ DMServer
    participant GW as 🎛️ Gateway

    User->>WKIM: SEND 包（群聊消息，mention.uids=[bot_uid]）
    WKIM->>WKSock: RECV 包（channelType=2，包含 mention 字段）
    WKSock->>Plugin: onMessage（BotMessage, payload.mention.uids）
    
    Plugin->>Plugin: 检查 mention.uids 是否包含 botUid
    
    alt 未被 @
        Plugin->>Plugin: 存入 groupHistories 滑动窗口（最多 historyLimit 条）
        Note over Plugin: 返回，不触发 AI
    else 被 @ 了
        Plugin->>Plugin: 从内存缓存取历史消息
        
        alt 缓存不足（< historyLimit/2）
            Plugin->>DMServer: POST /v1/bot/messages/sync {channel_id, limit}
            DMServer-->>Plugin: 历史消息列表（payload 为 base64）
            Plugin->>Plugin: 解码 base64 payload → JSON
        end
        
        Plugin->>Plugin: 构建历史上下文 JSON，注入消息前缀
        Plugin->>DMServer: GET /v1/bot/groups/{group_no}/members（刷新成员缓存）
        DMServer-->>Plugin: members[]（name, uid, role）
        Plugin->>Plugin: 更新 memberMap（name→uid）、uidToNameMap（uid→name）
        
        Plugin->>GW: 触发 AI 推理（含历史上下文）
        GW->>GW: AI 生成回复，包含 @username
        GW-->>Plugin: deliver({text: "@用户名 回复内容"})
        
        Plugin->>Plugin: 解析回复中的 @name → uid（通过 memberMap）
        Plugin->>DMServer: POST /v1/bot/sendMessage {mention: {uids: ["user_uid"]}}
        Note over DMServer: WuKongIM 推送消息给群组，@对应用户收到通知
    end
```

### @mention 双向解析

```
【入站解析（WuKongIM → AI）】
payload.mention.uids = ["user_uid_1", "user_uid_2"]
        ↓
uidToNameMap 查找 uid → name
        ↓
注入历史上下文时使用 @name 格式

【出站解析（AI → WuKongIM）】
AI 回复: "@张三 你好！"
        ↓
memberMap 查找 name → uid
        ↓
POST sendMessage { 
  content: "@张三 你好！",
  mention: { uids: ["uid_zhangsan"] }
}

【未解析时强制刷新】
GET /v1/bot/groups/{id}/members → 最新成员列表 → 重试解析
```

## 错误处理备选路径

```mermaid
flowchart TD
    A[AI 处理请求] --> B{成功?}
    B -->|是| C[流式回复]
    B -->|否| D[发送错误提示消息]
    
    C --> E{流式传输正常?}
    E -->|是| F[stream/end，冻结消息]
    E -->|否| G[降级：一次性 deliver 完整文本]
    
    H[WebSocket 断连] --> I[3秒后自动重连]
    I --> J[重新 DH 密钥交换]
    
    K[IM Token 过期（被踢）] --> L[POST /v1/bot/register?force_refresh=true]
    L --> M[获取新 im_token]
    M --> N[updateCredentials 重连]
```
