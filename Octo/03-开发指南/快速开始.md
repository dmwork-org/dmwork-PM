# 快速开始

> 5 分钟跑起来一个完整的 DMWork 开发环境。

[[开发环境搭建|下一步：详细开发环境搭建 →]]

## 概述

本指南帮助你快速启动 DMWork 后端服务（`dmworkim`）和 Web 客户端（`dmwork-web`）的开发环境。

## 前置条件

| 工具 | 最低版本 | 用途 |
|------|----------|------|
| Go | 1.20+ | dmworkim 后端编译运行 |
| Node.js | 22+ | dmwork-web / dmwork-adapters 前端 |
| Yarn | 1.22+ | Web 包管理（monorepo） |
| Docker | 24+ | 依赖服务（MySQL、Redis、WuKongIM、MinIO） |
| Docker Compose | 2.x | 一键启动依赖栈 |
| Git | 2.x | 代码拉取 |

```bash
# 验证版本
go version          # go1.20+ required
node --version      # v22+ required
docker --version    # 24+ required
```

## 克隆仓库

```bash
# 主服务器
git clone https://github.com/dmwork-org/dmworkim.git
cd dmworkim

# Web 客户端（可选）
git clone https://github.com/dmwork-org/dmwork-web.git

# AI 适配器（可选）
git clone https://github.com/dmwork-org/dmwork-adapters.git
```

## 启动后端（dmworkim）

### 第一步：启动依赖服务

```bash
cd dmworkim

# 启动 MySQL + Redis + WuKongIM + MinIO
make env-test
# 等价于: docker-compose -f docker-compose.test.yml up -d
```

确认服务启动：

```bash
docker ps
# 应看到以下容器运行中：
# - mysql:8.0
# - redis:7
# - wukongim
# - minio
```

### 第二步：配置文件

```bash
cp configs/tsdd.yaml.example configs/tsdd.yaml
```

最小配置（其他保持默认即可）：

```yaml
# configs/tsdd.yaml
mode: "debug"

db:
  mysql:
    addr: "127.0.0.1:3306"
    username: "root"
    password: "root"
    dbName: "dmwork"

redis:
  addr: "127.0.0.1:6379"

wukongim:
  apiUrl: "http://127.0.0.1:5001"
  wsUrl:  "ws://127.0.0.1:5100"

minio:
  endpoint:  "127.0.0.1:9000"
  accessKey: "minioadmin"
  secretKey: "minioadmin"
```

### 第三步：运行服务

```bash
# 开发模式启动（自动 SQL 迁移）
make run-dev

# 或者：
go run main.go
```

服务启动成功后：

```
[GIN-debug] Listening and serving HTTP on :8080
```

验证：

```bash
curl http://localhost:8080/v1/health
# {"status":"ok"}
```

## 启动 Web 客户端（dmwork-web）

```bash
cd dmwork-web

# 安装依赖
yarn install

# 启动开发服务器
yarn dev
# 或: turbo dev

# 访问 http://localhost:3000
```

## 启动 AI 适配器（dmwork-adapters）

```bash
cd dmwork-adapters/packages/openclaw-channel-dmwork

# 安装依赖
npm install

# 构建（TypeScript ESM）
npm run build
```

适配器通过 OpenClaw Plugin 配置使用，参见 [[OpenClaw适配器|05-适配层/OpenClaw适配器]]。

## 一键运行（make 快捷命令）

| 命令 | 说明 |
|------|------|
| `make run-dev` | 启动开发服务器（含配置加载） |
| `make env-test` | 启动 Docker 依赖服务 |
| `make test` | 运行所有单元测试 |
| `make build` | 编译生产二进制 |
| `make lint` | 代码检查 |

## 验证安装

```bash
# 1. 健康检查
curl http://localhost:8080/v1/health

# 2. 注册测试用户
curl -X POST http://localhost:8080/v1/user/usernameregister \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"Test1234!"}'

# 3. 登录
curl -X POST http://localhost:8080/v1/user/usernamelogin \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"Test1234!"}'
```

## 常见问题

**MySQL 连接失败**  
等待 Docker 容器完全启动（约 10s），或检查端口 3306 是否被占用。

**WuKongIM 未启动**  
运行 `docker-compose -f docker-compose.test.yml logs wukongim` 查看日志。

**端口冲突**  
修改 `configs/tsdd.yaml` 中的 `httpAddr` 字段（默认 `:8080`）。
