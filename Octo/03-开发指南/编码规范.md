# 编码规范

> DMWork 项目全栈编码规范：Go 模块约定 + TypeScript/Web Token 系统。

[[开发环境搭建|← 开发环境搭建]]

## 概述

DMWork 包含两个主要技术栈，各有独立规范：

- **Go（dmworkim）**：模块文件约定、目录结构、服务接口设计
- **TypeScript/React（dmwork-web、dmwork-adapters）**：Token 系统、组件规范、禁止硬编码

## Go 编码规范（dmworkim）

### 模块文件命名约定

每个业务模块位于 `modules/<name>/`，**必须遵循固定的文件命名**：

| 文件名 | 职责 | 是否必须 |
|--------|------|----------|
| `1module.go` | 模块注册（`init()` + `register.AddModule`） | ✅ 必须 |
| `api*.go` | HTTP 路由处理器（Gin Handler）、路由注册 | ✅ 必须 |
| `service.go` | 业务逻辑层（实现 `IService` 接口） | ✅ 必须 |
| `db*.go` | 数据库访问层（SQL Builder / dbr 操作） | ✅ 必须 |
| `model.go` | 请求/响应数据模型（Request/Response struct） | ✅ 必须 |
| `sql/` | SQL 迁移文件目录（`000001_init.sql` 格式） | ✅ 必须 |
| `event.go` | 事件处理器（可选，处理异步事件） | 按需 |
| `handler.go` | WuKongIM Webhook 处理（可选） | 按需 |

### 模块注册标准模板

```go
// modules/xxx/1module.go
package xxx

import (
    "github.com/dmwork-org/dmworkim/pkg/register"
    "github.com/dmwork-org/dmwork-lib/config"
)

func init() {
    register.AddModule(func(ctx interface{}) register.Module {
        x := New(ctx.(*config.Context))
        return register.Module{
            Name:     "xxx",
            SetupAPI: func() register.APIRouter { return x },
            SQLDir:   register.NewSQLFS(sqlFS), // embed SQL迁移
            Swagger:  swaggerContent,            // embed OpenAPI（可选）
            Service:  x.service,                 // 跨模块服务注册（可选）
            Start:    func() error { return nil }, // 启动钩子（可选）
            Stop:     func() error { return nil }, // 停止钩子（可选）
        }
    })
}
```

### 目录结构示例

```
modules/botfather/
├── 1module.go       ← 模块注册入口（命名必须以 1 开头确保 init 顺序）
├── api.go           ← Gin 路由定义
├── api_bot.go       ← Bot 相关 API Handler
├── api_userbot.go   ← 用户 Bot API Handler
├── service.go       ← IService 实现
├── db.go            ← 数据库访问
├── db_bot.go        ← Bot 表查询
├── model.go         ← 数据模型（Request/Response）
├── event.go         ← 事件监听（可选）
└── sql/
    ├── 000001_init.sql
    ├── 000002_add_stream.sql
    └── ...
```

### IService 接口约定

每个模块导出 `IService` 接口，供其他模块跨模块调用（**避免直接 import 实现层**）：

```go
// modules/user/service.go
type IService interface {
    GetUser(uid string) (*resp.UserResp, error)
    GetUsers(uids []string) ([]*resp.UserResp, error)
    IsFriend(uid, toUID string) (bool, error)
    // ... 其他方法
}
```

跨模块依赖通过 `source` 模块的 Provider 接口注册，**严禁直接 import 其他模块的实现包**。

### 错误处理

```go
// ✅ 推荐：使用 pkg/errors 包装
import "github.com/pkg/errors"
return errors.Wrap(err, "failed to get user")

// ✅ 推荐：业务错误使用 wkhttp.Error
c.ResponseError(errors.New("用户不存在"))

// ❌ 禁止：直接 panic（除非初始化阶段）
panic(err)
```

### SQL 迁移文件规范

```sql
-- 000001_init.sql
-- +migrate Up
CREATE TABLE IF NOT EXISTS `xxx` (
    `id`         bigint(20)   NOT NULL AUTO_INCREMENT,
    `uid`        varchar(40)  NOT NULL DEFAULT '',
    `created_at` timestamp    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at` timestamp    NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_uid` (`uid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- +migrate Down
DROP TABLE IF EXISTS `xxx`;
```

## TypeScript/Web 编码规范（dmwork-web）

### Token 系统 — 禁止硬编码颜色/间距

这是 `dmwork-web` 最重要的规范，来自 `AGENTS.md`：

```typescript
// ❌ 严格禁止：硬编码颜色
style={{ color: '#7C5CFC' }}
style={{ padding: '16px' }}
style={{ borderRadius: '8px' }}

// ✅ 正确：使用 CSS Token 变量
style={{ color: 'var(--wk-brand-primary)' }}
style={{ padding: 'var(--wk-sp-4)' }}
style={{ borderRadius: 'var(--wk-r-md)' }}
```

### 颜色 Token 优先级

```
1. --semi-color-*   → Semi Design 系统色（自动暗色模式支持）
2. --wk-brand-*     → 品牌色（紫/青渐变）
3. --wk-text-*      → 文字色补充
4. --wk-border-*    → 边框色补充
5. --wk-ai-*        → AI 专属色（AI 消息背景/边框）
```

### 间距系统（4px 栅格）

```css
--wk-sp-1: 4px    /* 最小间距 */
--wk-sp-2: 8px
--wk-sp-3: 12px
--wk-sp-4: 16px   /* 标准间距 */
--wk-sp-5: 20px
--wk-sp-6: 24px
--wk-sp-8: 32px
--wk-sp-10: 40px
--wk-sp-12: 48px  /* 大间距 */
```

### 圆角系统

```css
--wk-r-xs:   4px    /* 标签/Badge */
--wk-r-sm:   6px    /* AI Bot 头像（方形）*/
--wk-r-md:   10px   /* 群组头像、按钮 */
--wk-r-lg:   14px   /* 大卡片 */
--wk-r-xl:   20px   /* 消息气泡 */
--wk-r-full: 9999px /* 人类用户头像（圆形）*/
```

### 身份识别规则（严格执行）

```
人类用户  → border-radius: 50%（--wk-r-full）
AI Bot   → border-radius: var(--wk-r-sm)  ← 6px 方形，视觉区分！
群组     → border-radius: var(--wk-r-md)
```

### 组件规范

**Provider 模式（状态管理）**：

```typescript
// ✅ 使用 Provider + ProviderListener，不引入 Redux/MobX
class MyVM extends ProviderListener {
    data: string[] = []
    
    async load() {
        this.data = await fetchData()
        this.notifyListener()  // 触发 React 重渲染
    }
}

// 组件中使用
<Provider create={() => new MyVM()} render={(vm: MyVM) => (
    <div>{vm.data.map(item => <span key={item}>{item}</span>)}</div>
)} />
```

**EndpointManager 扩展点**：

```typescript
// ✅ 新功能通过 EndpointManager 注册，不直接修改核心组件
WKApp.endpointManager.setMethod("chat.toolbars.myFeature", 
    () => <MyToolbarButton />,
    { category: EndpointCategory.chatToolbars, sort: 100 }
)
```

### TypeScript 规范

```typescript
// ✅ 使用 interface，不用 type（除非联合类型）
interface UserInfo {
    uid: string
    name: string
    avatar?: string
}

// ✅ 明确返回类型
function fetchUser(uid: string): Promise<UserInfo> { ... }

// ❌ 避免 any（除非必要的第三方库交互）
const data: any = response.data  // 禁止

// ✅ 使用泛型
const data: UserInfo = response.data as UserInfo
```

### ESLint 配置

所有包继承 `packages/eslint-config-custom`，关键规则：

- `@typescript-eslint/no-explicit-any`: error
- `react-hooks/rules-of-hooks`: error
- `react-hooks/exhaustive-deps`: warn
- `no-hardcoded-colors`（自定义规则）：error

## Git 提交规范

```
feat: 新功能
fix: Bug 修复
docs: 文档更新
refactor: 重构（不影响功能）
test: 测试相关
chore: 构建工具/依赖更新
```

示例：
```
feat(botfather): add stream message support
fix(webhook): correct VIVO push signature algorithm
docs(api): update BotRegisterResp with owner_channel_id field
```
