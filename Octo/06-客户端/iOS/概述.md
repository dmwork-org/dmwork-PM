# DMWork iOS 客户端概述

> Objective-C + CocoaPods 模块化 IM 客户端，最低支持 iOS 14.0。

[[概述|← 客户端概述]]

## 概述

`dmwork-ios` 是 DMWork 的 iOS 客户端，基于悟空IM（WuKongIM）iOS SDK 构建，采用模块化 CocoaPods 架构。

**关键标识**：
- **Bundle ID**：`com.dmwork.im`（生产）
- **productName**：`LiMaoIMDemo`（遗留历史品牌名，仅出现在 Xcode 工程配置中）
- **最低 iOS 版本**：iOS 14.0（CORRECTED：非 13.0）
- **语言**：Objective-C（主要）+ 部分 Swift（较少）

> ⚠️ **CORRECTED**：`productName=LiMaoIMDemo` 是从 LiMao IM 前身 fork 的遗留标识，实际产品已是 DMWork；iOS 最低版本为 **14.0**，非 13.0。

## 整体架构

```
TangSengDaoDaoiOS.xcworkspace
│
├── TangSengDaoDaoiOS（主 Target）
│   ├── AppDelegate.m            ← 应用入口，初始化所有配置
│   ├── WKMainTabController.m    ← 主 TabBar（3个Tab）
│   └── WKMeVC                  ← "我的"页面
│
├── Pods（CocoaPods 管理的第三方依赖）
│
└── Modules（本地 Pod 模块）
    ├── WuKongIMiOSSDK/          ← 悟空IM 核心 SDK（通讯底层）
    ├── WuKongBase/              ← 基础功能模块（UI + 网络 + 服务）
    ├── WuKongLogin/             ← 登录注册模块
    ├── WuKongContacts/          ← 联系人模块
    └── WuKongDataSource/        ← 数据源模块
```

### 数据流向

```
用户操作 → ViewController → ViewModel → 
    WKAPIClient（HTTP）→ dmworkim 业务层
    WKSDK（WebSocket）→ WuKongIM 通讯层
```

## 核心模块详解

### WuKongIMiOSSDK — IM 底层 SDK

`Modules/WuKongIMiOSSDK/` — 对应 WuKongIM 官方开源 iOS SDK。

**关键单例**：`WKSDK`，持有所有通讯管理器：

| 管理器 | 职责 |
|--------|------|
| `connectionManager` | WebSocket 长连接建立、重连、断线 |
| `chatManager` | 消息发送、接收、消息列表管理 |
| `channelManager` | 频道信息缓存与管理 |
| `conversationManager` | 最近会话列表管理 |
| `mediaManager` | 媒体文件上传下载 |
| `receiptManager` | 消息已读回执 |
| `cmdManager` | CMD 消息处理（控制类消息） |
| `coder` | 消息编解码（加密/解密） |
| `bodyCoderManager` | Packet Body 自定义编解码 |

**加密工具**：
- `WKAESUtil`：AES 对称加密
- `WKRSAUtil`：RSA 非对称加密（密钥交换）
- 支持端到端加密（Signal 协议）

**协议特点**：WuKongIM 使用私有二进制协议（非标准 MQTT/WebSocket 文本），通过 TCP/WebSocket 传输自定义帧，`WKCoder` 负责帧编解码。

### WuKongBase — 基础功能模块

`Modules/WuKongBase/` — 最重的业务模块，包含大部分 IM UI 和服务层。

**目录结构**：

```
WuKongBase/Classes/
├── Sections/               ← 业务 UI 分区
│   ├── Messages/           ← 消息气泡（文字/图片/语音/GIF/文件/名片/合并转发）
│   ├── ConversationList/   ← 最近会话列表
│   ├── Conversation/       ← 聊天页面相关
│   ├── ConversationSetting/← 聊天设置（免打扰/置顶/群公告等）
│   ├── Contacts/           ← 联系人相关 UI
│   ├── Group/              ← 群组管理 UI
│   ├── Me/                 ← 个人信息页
│   ├── Scan/               ← 扫一扫
│   ├── Channel/            ← 频道相关
│   ├── Common/             ← 公共组件
│   └── InlineQuery/        ← 内联查询（AI Bot 行内搜索）
├── Model/                  ← 数据模型
├── Services/               ← 服务层
│   ├── Base/               ← WKAPIClient、WKNetworkListener
│   ├── Core/               ← WKCore、WKEndpointManager（端点总线）
│   └── Category/           ← ObjC Category 扩展
└── Vendor/                 ← 内嵌第三方库（Bugly 等）
```

**关键服务**：
- `WKAPIClient`：AFNetworking + PromiseKit 封装，GET/POST/PUT/DELETE/文件上传，统一 token 注入和错误处理
- `WKEndpointManager`：模块间事件总线，端点名称解耦模块调用
- `WKSyncService`：数据同步服务，本地与服务端增量同步

**消息 Cell 类型**：

| Cell 类 | 消息类型 |
|---------|----------|
| `WKTextMessageCell` | 文本消息 |
| `WKImageMessageCell` | 图片消息 |
| `WKVoiceMessageCell` | 语音消息 |
| `WKFileMessageCell` | 文件消息 |
| `WKGIFMessageCell` | GIF 表情 |
| `WKCardCell` | 名片消息 |
| `WKMergeForwardCell` | 合并转发 |
| `WKSystemMessageCell` | 系统消息 |
| `WKTypingMessageCell` | 正在输入提示 |

**MVVM 模式**：每个 `ViewController` 配套 `ViewModel`（如 `WKConversationListVC` + `WKConversationListVM`），VM 持有 ReactiveObjC Subject 驱动 UI 更新。

### WuKongLogin — 登录模块

`Modules/WuKongLogin/`

支持的登录方式：
- 手机号验证码登录
- 用户名/密码登录
- OAuth 第三方授权
- 二维码扫码登录（WuKongIM advance auth 机制）
- 邮箱登录

### WuKongContacts — 联系人模块

`Modules/WuKongContacts/`

功能：
- 联系人列表（字母索引 + 分组）
- 好友申请/确认
- 群组管理 UI
- 搜索联系人
- Space 感知（Space 切换时重新同步）

### WuKongDataSource — 数据源模块

`Modules/WuKongDataSource/`

数据源模块实现 `WuKongBase` 中定义的数据接口，处理：
- 频道数据同步
- 消息本地缓存
- Space 成员同步

## 第三方依赖（关键）

| 依赖 | 引入方式 | 用途 |
|------|----------|------|
| AFNetworking | CocoaPods | HTTP 网络层 |
| PromiseKit | CocoaPods | 异步 Promise 链式 |
| ReactiveObjC | CocoaPods | MVVM 响应式绑定 |
| Masonry | CocoaPods | AutoLayout DSL |
| SDWebImage | CocoaPods | 图片异步加载/缓存 |
| **Bugly** | **vendored_frameworks** | 崩溃分析（非 CocoaPods！） |

> ⚠️ **CORRECTED**：Bugly 通过 `vendored_frameworks` 内嵌引入（`Vendor/` 目录下的 `.framework`），**不是** 通过 CocoaPods 安装。这是与文档描述不符的重要细节。

## 端点驱动架构

iOS 端同样采用 `WKEndpointManager`（端点总线）实现模块解耦：

```objc
// 注册端点
[[WKEndpointManager shared] setMethod:@"show_chat" 
                               handler:^(id param) {
    // 展示聊天页面
    WKChatVC *vc = [[WKChatVC alloc] initWithChannel:param];
    [self.navigationController pushViewController:vc animated:YES];
}];

// 调用端点
[[WKEndpointManager shared] invoke:@"show_chat" param:channel];
```

## 主题与深色模式

```objc
// AppDelegate.m
UIWindow.overrideUserInterfaceStyle = 
    WKApp.shared.config.style == WKThemeStyleDark 
    ? UIUserInterfaceStyleDark 
    : UIUserInterfaceStyleLight;
```

通过 `WKApp.shared.config.style` 动态设置全局界面样式，全局支持深色/浅色模式切换，iOS 13+ 特性（但最低部署目标为 iOS 14.0）。
