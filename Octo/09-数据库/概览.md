# 数据库概览

DMWorkim 使用 MySQL 数据库，共 **62 张表**，按模块分组。

> **技术栈**：MySQL + gocraft/dbr（SQL Builder）+ rubenv/sql-migrate（迁移管理）

## 模块表统计

| 模块 | 表数量 | 主要职责 |
|------|--------|---------|
| [[#base]] | 2 | 应用基础信息、事件系统 |
| [[#botfather]] | 1 | Bot API Key 管理 |
| [[#channel]] | 1 | 频道设置 |
| [[#common]] | 5 | 通用配置、版本管理、序号发号 |
| [[#group]] | 5 | 群组及成员管理 |
| [[#message]] | 15 | 消息核心模型（含分片、扩展、偏移） |
| [[#report]] | 2 | 举报系统 |
| [[#robot]] | 2 | 机器人管理 |
| [[#space]] | 3 | 团队空间（多租户） |
| [[#user]] | 15 | 用户体系（含好友、设备、三方登录） |
| [[#webhook]] | 5 | 消息分片存储（message~message4） |
| [[#workplace]] | 6 | 工作台应用管理 |
| **合计** | **62** | |

## 模块详情

### base

| 表名 | 说明 |
|------|------|
| `app` | 应用身份标识，存储接入系统的 App 信息（AppID/AppKey/AppName/AppLogo） |
| `event` | 内部事件异步队列，cron 59 秒轮询投递给 WuKongIM |

### botfather

| 表名 | 说明 |
|------|------|
| `user_api_key` | 用户 API Key（uk_ 前缀），BotFather OpenAI 兼容接口认证 |

### channel

| 表名 | 说明 |
|------|------|
| `channel_setting` | 频道设置，存储父子关系（parent_channel_id）和消息自动删除配置 |

### common

| 表名 | 说明 |
|------|------|
| `app_version` | App 版本管理（iOS/Android，是否强制升级） |
| `app_config` | 系统全局配置（RSA 密钥、超级 Token、功能开关等） |
| `seq` | 序号发号器（分布式 ID，key/min_seq/step） |
| `chat_bg` | 聊天背景图片库 |
| `app_module` | App 功能模块配置（控制前端功能开关） |

### group

| 表名 | 说明 |
|------|------|
| `group` | 群组主表（群名、创建者、状态、设置、space_id） |
| `group_member` | 群成员表（角色、禁言时长、是否机器人） |
| `group_setting` | 用户对群的个性化设置（免打扰、置顶、阅后即焚等） |
| `group_invite` | 群邀请头部记录（整次邀请操作） |
| `invite_item` | 群邀请明细（每个被邀请者一条记录） |

### message

| 表名 | 说明 |
|------|------|
| `send_history` | 管理员代发消息记录 |
| `message_extra` | 消息扩展（撤回、编辑、已读数、置顶） |
| `member_readed` | 成员已读列表 |
| `conversation_extra` | 会话扩展（浏览进度、草稿） |
| `reaction_users` | 消息 emoji 回应 |
| `message_user_extra` | 用户对消息的个人扩展·分片0（语音已读、单向删除） |
| `message_user_extra1` | 用户对消息的个人扩展·分片1 |
| `message_user_extra2` | 用户对消息的个人扩展·分片2 |
| `channel_offset` | 频道消息读取偏移·分片0 |
| `channel_offset1` | 频道消息读取偏移·分片1 |
| `channel_offset2` | 频道消息读取偏移·分片2 |
| `device_offset` | 设备消息偏移量（每设备每频道的读取进度） |
| `user_last_offset` | 用户消息最新偏移量 |
| `reminders` | 消息提醒（@我、草稿等） |
| `reminder_done` | 提醒完成记录 |

> 注：`prohibit_words`（违禁词）和 `pinned_message`（置顶消息）也属于 message 模块。实际 message 模块含表数含上述所有。

### report

| 表名 | 说明 |
|------|------|
| `report_category` | 举报类别（支持多级分类） |
| `report` | 举报记录 |

### robot

| 表名 | 说明 |
|------|------|
| `robot` | 机器人主表（robot_id 对应 user.uid，Bot = 特殊用户） |
| `robot_menu` | 机器人菜单/命令表 |

### space

| 表名 | 说明 |
|------|------|
| `space` | 团队空间主表（多租户隔离单元） |
| `space_member` | 空间成员（含角色：Owner/Admin/Member） |
| `space_invitation` | 空间邀请码（含过期时间、使用次数限制） |

### user

| 表名 | 说明 |
|------|------|
| `user` | 用户主表（真实用户 + 机器人 + 系统账号，robot 字段区分） |
| `user_setting` | 用户对他人的个性化设置（每对关系一条） |
| `device` | 用户设备表 |
| `friend` | 好友关系表（已废弃，迁移到 user_setting） |
| `user_deny` | 拒绝好友申请记录 |
| `shortno` | 短编号（用户短码，通过 pkg/common 的 IService 管理） |
| `apply` | Bot 好友申请记录（bot_apply 语义） |
| `bot_apply` | Bot 申请详细信息 |
| `app`（user 模块视角）| 见 base.app |
| `verify_code` | 短信/邮箱验证码 |
| `gitee_user` | Gitee OAuth 关联 |
| `github_user` | GitHub OAuth 关联 |
| `wx_user` | 微信 OAuth 关联 |
| `pc_login` | PC 端登录状态 |

### workplace

## 迁移管理

数据库迁移文件嵌入在各模块 SQL 目录中（Go `//go:embed` 指令），通过 `rubenv/sql-migrate` 管理：

```
modules/<module>/sql/
  ├── <module>-<yyyymmdd>-01.sql
  ├── <module>-<yyyymmdd>-02.sql
  └── ...
```

模块初始化时（`module.Setup`）自动执行未运行的迁移文件，按文件名排序。
