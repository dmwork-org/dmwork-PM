## 进行中（2026-03 活跃开发）

### 🔄 Space per-API-Key 隔离

**背景**：原有 API Key 无 Space 绑定，Bot 可跨 Space 访问，缺乏隔离。

**进展**：
```sql
-- 已在 SQL 迁移文件中发现
ALTER TABLE user_api_key ADD COLUMN space_id VARCHAR(40);
ALTER TABLE user_api_key ADD UNIQUE INDEX idx_uid_space (uid, space_id);
```

**目标**：每个 Bot 在每个 Space 中有独立的 API Key，实现 OpenAI 兼容接口的 Space 级隔离。

**参见**：[[ADR-002-Space前缀约定]] | [[Space多租户]]

---

### 🔄 流式消息优化

**背景**：当前流式消息依赖 `stream/start → sendMessage(stream_no) → stream/end` 三步流程，存在潜在竞态条件。

**已知优化方向**：
- 流式传输失败时自动降级（`streamFailed = true` → 一次性发送完整文本）
- 断线重连时流式状态恢复
- 客户端渲染性能优化（频繁 DOM 更新）

---

### 🔄 群聊 Bot 标识

**背景**：群聊中 Bot 与人类用户的视觉区分不够明显。

**已知方向**：
- 消息列表中 Bot 头像角标
- `robot=1` 字段在 API 响应中的前端消费
- 群成员列表中 Bot 标识

## 规划中（待立项）

### 📋 Bot 市场 / 应用商店

- 类似 BotFather 的 Web UI
- Bot 上架、审核流程
- Bot 分类、搜索、评分

### 📋 Webhook 数据源扩展

- 当前 `webhook` 模块支持 datasource 绑定
- 规划扩展为更通用的 AI 触发器（定时、外部事件驱动）

### 📋 消息搜索增强

- 当前支持基础消息搜索（`/v1/message/search`）
- Elasticsearch 集成已在 `base/elastic` 模块中有基础
- 规划：全文搜索、语义搜索

### 📋 国际化 / 多语言

- 当前 UI 以中文为主
- 规划：英文版本，国家/地区配置（`/v1/common/countries` 已有）

### 📋 消息加密增强

- Signal Protocol 支持已在代码中有标记（`signal INT DEFAULT 0`）
- 规划：端到端加密的完整 UI 流程
