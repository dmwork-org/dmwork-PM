# 适配层概述

`dmwork-adapters` 是一个 **AI 适配器集合**，让不同的 AI Agent 框架能通过 DMWork 即时通讯平台（底层使用 WuKongIM）收发消息。

## 核心问题

WuKongIM 使用**自定义二进制协议**进行实时消息传输，并在握手阶段进行 **DH 密钥交换 + AES 加密**。AI Agent 框架（OpenClaw、Claude SDK）对 IM 协议一无所知，无法直接连接。

两个适配器都解决同一问题：在 AI 框架与 DMWork WuKongIM 服务器之间建立实时双向通信桥梁。

## 两个适配器

| 适配器 | 目标框架 | 形态 | 说明 |
|--------|----------|------|------|
| `openclaw-channel-dmwork` | OpenClaw AI 框架 | npm 包（Channel Plugin） | 深度集成框架生命周期 |
| `claude-code-dmwork-ws` | Claude Agent SDK | 独立 Node.js 网关进程 | 轻量独立进程 |

## 共同能力

两个适配器都处理：

```mermaid
graph LR
    subgraph 适配器
        proto[WuKongIM 二进制协议编解码]
        dh[DH-AES 端对端加密握手]
        route[消息路由\nDM vs 群聊]
        mention[@mention 检测与解析]
        stream[流式消息支持]
        reconnect[断线重连与 Token 刷新]
    end
    AI[AI Agent 框架] <--> 适配器
    适配器 <--> WK[WuKongIM WS]
    适配器 <--> DM[DMWork REST API]
```

## 架构对比

### openclaw-channel-dmwork 架构

```
OpenClaw Framework
├── channel.ts      (插件入口 + 出站处理)
├── inbound.ts      (入站消息处理管道 - 11步)
├── socket.ts       (WuKongIM 二进制协议 + DH/AES)
├── api-fetch.ts    (DMWork REST API 调用)
├── mention-utils.ts(@mention 解析工具)
├── config-schema.ts(配置 Schema 声明)
├── accounts.ts     (账号解析)
└── runtime.ts      (OpenClaw 运行时引用)
```

### claude-code-dmwork-ws 架构

```
独立 Node.js 进程
├── gateway.ts      (核心网关)
├── dmwork/socket.ts(与 openclaw 版相同的协议实现)
├── dmwork/api.ts   (DMWork REST API)
├── session-store.ts(对话历史 JSON 持久化)
└── group-context.ts(群聊缓存)
```

## 技术栈

| 类别 | 依赖 | 用途 |
|------|------|------|
| 加密 | `curve25519-js` | Curve25519 ECDH 密钥交换 |
| 加密 | `crypto-js` | AES-CBC 加解密 |
| 加密 | `md5-typescript` | shared secret → AES key 派生 |
| WebSocket | `ws` | WebSocket 客户端 |
| 语言 | TypeScript 5.9 | 类型安全开发 |
| 测试 | `vitest` | 单元测试框架 |
