## 二、任务需求

### 2.1 背景

当前 Bot API 有以下群组相关能力：
- `GET /v1/bot/groups` — Bot 获取所在群组列表
- `GET /v1/bot/groups/:group_no` — Bot 获取群组信息
- `GET /v1/bot/groups/:group_no/members` — Bot 获取群成员列表

但 **Bot 无法主动邀请其他 Bot 加入群聊**。在龙虾协作场景下，一个 Bot 需要拉其他 Bot 进群组建协作团队，目前只能靠人类手动操作。

### 2.2 目标

实现 Bot API 支持邀请其他 Bot 进群，让 Bot 具备自主组建协作团队的能力。

### 2.3 需要回答的设计问题

以下问题需要在设计方案中明确回答：

1. **权限模型**
   - 所有 Bot 都能邀请，还是只有群主/管理员 Bot 能邀请？
   - Bot 能邀请人类用户吗？还是只能邀请其他 Bot？
   - 被邀请的 Bot 需要"同意"吗？还是直接拉进群？
   - 如果被邀请的 Bot 不在线怎么办？

2. **API 设计**
   - 端点路径：`POST /v1/bot/groups/:group_no/invite` 还是其他？
   - 请求体格式：邀请单个 Bot 还是支持批量？
   - 响应格式：同步返回结果还是异步通知？
   - 错误场景：Bot 已在群中、Bot 不存在、无权限等

3. **与现有系统的交互**
   - 群成员管理（group 模块）：现有 `POST /v1/groups/:group_no/members` 是用户 JWT 鉴权的，Bot Token 鉴权怎么走？
   - WuKongIM：新成员加入后需要订阅频道，Bot 的订阅流程是什么？
   - 事件通知：被邀请的 Bot 怎么知道自己被拉进了新群？通过 events API 推送？

4. **群聊内展示**
   - 邀请成功后是否在群里发一条系统消息（"Bot A 邀请 Bot B 加入群聊"）？
   - 消息类型是什么？ContentType 编号？
