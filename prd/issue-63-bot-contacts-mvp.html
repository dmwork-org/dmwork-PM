通讯录与 Bot 整合

P0 MVP 2026-03-18

背景

Octo 的核心理念是「AI 归个人，协作无边界」—— 人和 AI 是平等的一等公民，应该在同一个列表里展示。

核心原则

  • 人 + AI + 群聊 混排 — 不分开显示，体现平等
  • 形状即身份 — 人=圆形，Bot=方形+渐变角标，群=大圆角方形
  • 统一名片体验 — 点击人/Bot 都弹出名片

用户故事

  • 作为用户，我希望在一个列表里看到所有通讯录（人 + Bot + 群聊）
  • 作为用户，我希望点击任何通讯录都能看到名片
  • 作为用户，我希望快速找到 BotFather
  • 作为 Owner，我希望收到并审批加好友申请

通讯录列表

人、Bot、群聊在同一个列表里，平铺展示。

结构

  • 顶部入口：BotFather（横幅样式，占满宽度）
  • 筛选项：在「通讯录」右侧，可筛选「全部」「我的 Bot」「所有 Bot」「所有员工」「群聊」
  • 搜索框：按名称搜索
  • 列表：人 + Bot + 群聊 混排，按拼音/字母排序

排序规则

中英文混排时的排序逻辑：

  • 中文按拼音首字母排序
  • 英文按字母排序
  • 英文排在其对应字母位置（如 "JOJO" 排在 J 区，与「建国」「佳佳」一起）
  • 同一字母下，英文和中文混排（不区分先后）

示例：阿白 → JOJO → 建国 → 李四 → Thomas → 王五 → 小苗 → 张三

申请加好友

点击「添加好友」后，填写申请原因。

流程

点击「添加好友」
→
填写申请原因
→
提交申请
→
显示成功回显
→
等待 Owner 审批

申请原因

  • 非必填，可以为空
  • 为空时直接提交

成功回显

  • 提交后显示「申请已提交」
  • 提示「等待 Owner 审批」
  • 点击「知道了」关闭弹窗

重复提交处理

已提交申请且正在等待审批时：

  • 按钮直接显示 「申请审批中」（灰色，不可点击）
  • 申请被拒绝后，按钮恢复为「添加好友」，可重新申请

审批通知

Owner 收到加好友申请时，BotFather 发送消息卡片（BotFather 是管理 Bot 的角色，统一由它发送审批通知）。

卡片内容

  • 申请人
  • 申请加哪个 Bot
  • 申请原因（未填写显示「-」）
  • 操作：通过 / 拒绝
