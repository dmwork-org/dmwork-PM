PRD: AI 看板（Owner Dashboard）

优先级：P0 状态：Draft 作者：刘乐君 版本：v1.1

背景

Octo 的核心理念是「AI 归个人，协作无边界」—— 每个人拥有自己的 AI，可以被添加到多个 Space（组织/企业空间）与他人协作。

Owner 需要一个全局视角来了解自己 Bot 的使用情况，包括：谁在用、用了多少、在哪些 Space、工作轨迹等。同时需要保护用户隐私——管理权 ≠ 偷窥权。

用户故事

  • 作为 Owner，我希望看到 Bot 的全局使用统计（跨 Space）
  • 作为 Owner，我希望知道 Bot 在哪些 Space，每个 Space 有多少群
  • 作为 Owner，我希望看到用户和群聊的 Token 消耗排行
  • 作为 Owner，我希望查看 Bot 的工作轨迹（不含对话内容）
  • 作为 Owner，我希望了解 Bot 的响应速度和健康状态

核心原则

🔒 隐私保护

Owner 能看统计数据，不能看对话内容。管理权 ≠ 偷窥权。

看板主页

Owner 查看 Bot 的全局使用统计，包含跨 Space 的数据汇总。

  • 💬 消息（会话列表）
  • 👥 通讯录（联系人）

隐私边界

  • 只显示「发给谁」，不显示发送内容
  • 只显示「调用了什么工具」，不显示工具参数
  • Owner 能看到工作轨迹，看不到具体聊了什么

术语定义

术语定义单位
Token 消耗Bot 处理消息消耗的 Token 数量（含输入 + 输出）k (千 Token)
用户数与 Bot 有过私聊互动的好友数量人
群聊数Bot 所在的群聊数量个
平均响应时间从用户发送消息到 Bot 开始响应的平均时间秒 (s)
Space组织/企业空间，Bot 可以加入多个 Space个
最近活跃Bot 在该用户/群聊中最后一次发消息的时间—
工作日志Bot 的工作轨迹记录，不含对话内容—

隐私边界

✓ Owner 可以看
  • Bot 状态、版本信息
  • Token 消耗统计
  • 用户/群聊列表
  • 各用户/群聊的 Token 消耗
  • 最近活跃时间
  • 所在 Space 列表
  • 工作轨迹（调用了什么工具、发给谁）
✗ Owner 不能看
  • 他人与 Bot 的对话内容
  • 用户的个人敏感信息
  • 对话详情/聊天记录
  • 工具调用的具体参数
  • 消息的具体内容

核心原则：管理权 ≠ 偷窥权。Owner 只能看统计数据和工作轨迹，不能看对话内容。

边界条件

场景处理方式
Bot 当天没有消息显示 0，变化趋势显示「—」
新创建的 Bot只有当天数据，无变化趋势
Bot 被删除保留历史数据，标记「已删除」
数据延迟最多 1 小时，显示「数据更新中」
Bot 切换模型Token 累加统计，不区分模型
用户无私聊只有群聊数据时，用户列表不显示该用户
群聊被解散Bot 自动退出，历史数据保留
Bot 退出 Space该 Space 数据归档，不再累加
筛选范围无数据显示空状态「暂无数据」
